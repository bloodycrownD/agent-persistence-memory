# 方案设计

## 使用说明

1. 先读取需求文档：
   - `docs/Iterations/<需求名称>/spec.md`
2. 生成方案文档：
   - `docs/Iterations/<需求名称>/plan.md`
3. 方案必须面向实现，至少包含：
   - 总体实现思路与架构
   - 最终项目结构
   - 文件级改动点
   - 兼容性或迁移说明（如需要）
   - 分步骤实现计划
   - 测试策略与测试用例
   - 风险与回滚方案
4. 保证可执行性：每一步都应可落实、可验证。
5. 编码前先请求用户确认 `plan.md`。

## 文档模板

```markdown
# <需求名称> 设计方案

## 设计目标

## 总体方案

## 最终项目结构

## 变更点清单

## 详细实现步骤

## 测试策略
### 测试用例

## 风险与回滚方案
```
