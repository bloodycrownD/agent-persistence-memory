# 需求评审

## 使用说明

1. 使用 `AskQuestion` 与用户澄清需求细节。
2. 每次只问一个聚焦问题，直到关键信息明确：
   - 目标与业务价值
   - 范围（包含 / 不包含）
   - 约束与依赖
   - 验收标准
3. 澄清完成后，生成需求文档：
   - `docs/Iterations/<需求名称>/spec.md`
4. `spec.md` 至少包含以下内容：
   - 背景
   - 目标
   - 范围
   - 功能需求
   - 非功能需求
   - 验收标准
   - 风险与待确认项
5. 明确告知生成路径，并请用户进行最终确认。

## 文档模板

```markdown
# <需求名称> 需求说明

## 背景

## 目标

## 范围
### 包含范围
### 不包含范围

## 功能需求

## 非功能需求

## 验收标准

## 风险与待确认项
```
