# apm-read-full-view PRD

## 背景
当前 APM CLI 支持分别读写角色（role）、持久记忆（persist）和任务动态（dynamic）。但在 Agent 启动或上下文恢复场景下，用户或 Agent 需要一次性获取当前所有核心记忆的聚合视图，以快速建立认知。

## 目标（含成功指标）
实现 `apm read` 命令，将分散在不同文件中的记忆内容聚合输出。
- **成功指标**：
  - 支持一条命令输出完整的记忆上下文。
  - 输出格式清晰，符合 Markdown 规范，易于 Agent 理解。
  - 自动过滤元数据，仅展示有效正文。

## 用户与场景
- **Agent 初始化**：Agent 在开始工作前，通过 `apm read` 获取自己的角色设定、长期规则和当前任务进度。
- **人工审计**：开发者通过该命令快速检查当前记忆库的状态。

## 范围
### 包含范围
- **聚合输出**：汇总 `memory/role.md`、`memory/persist.md` 和 `memory/dynamic.md` 的正文内容。
- **格式化**：按“# 角色”、“# 持久记忆”、“# 动态记忆”三个二级标题分段。
- **内容清洗**：移除各文件顶部的 YAML front matter，仅保留 Markdown 正文。
- **空内容处理**：若某部分内容为空，则不显示对应的标题。

### 不包含范围
- **知识库 (kb) 聚合**：不包含 `kb/` 下的文档。
- **交互式编辑**：该命令仅负责读取。

## 核心需求
1. **聚合读取**：命令需依次读取 `memory/` 下的三个核心文件。
2. **正文提取**：必须正确解析并剔除 YAML front matter。
3. **结构化输出**：按约定顺序（角色 -> 持久记忆 -> 动态记忆）排列，并注入对应的二级标题。
4. **静默空项**：自动检测内容有效性，空内容不占位、不输出标题。

## 验收标准
### 基础输出
- **Given** 三个记忆文件均有正文内容。
- **When** 执行 `apm read`。
- **Then** 控制台依次输出三个二级标题及对应的正文，且不含 `---` 包围的元数据。

### 部分缺失
- **Given** 仅 `role.md` 有内容，`persist.md` 和 `dynamic.md` 为空。
- **When** 执行 `apm read`。
- **Then** 仅输出“# 角色”及其内容，不出现其他标题。

### 顺序一致性
- **Then** 输出顺序必须固定为：角色、持久记忆、动态记忆。
